# Current Landing Expansion Plan

## 1. Current Live Page Baseline

Current URL: https://fina-calle-landing.vercel.app/

Current headline:

SOFTWARE FOR BRANDS THAT MOVE DIFFERENT.

Current subline:

Interactive storefronts, QR experiences, mini-games, and brand systems for local businesses.

Current CTAs:

- Start a Project
- View Colattao
- View Menu

Current case study line:

Colattao Cafe Rush is the first live case study.

Current identity:

Fina Calle OS / AMMA Ventures LLC

## 2. Expansion Principle

The current page has the right seed and should be expanded, not replaced blindly. The headline should remain as the brand voice because it gives Fina Calle a sharper identity than generic agency language.

The expansion should clarify what Fina Calle OS builds, show Colattao as proof, and give future clients a clean path into intake without adding payment, backend, or portal complexity too early.

## 3. Keep Current Headline as Brand Voice

Keep:

SOFTWARE FOR BRANDS THAT MOVE DIFFERENT.

Use supporting copy below it to explain the offer in plain language:

Fina Calle OS builds interactive digital storefronts, QR experiences, mini-games, intake systems, and modular brand tools for local businesses.

## 4. Add: What Fina Calle OS Builds

Add a section that makes the product system concrete.

Recommended section goal:

Show that Fina Calle is not a generic website agency. It builds modular digital experiences that help local businesses look premium, engage customers, and grow into richer systems over time.

Suggested section title:

What Fina Calle OS Builds

Suggested supporting line:

Mobile-first modules for local businesses that want more than a static page.

## 5. Add Module Grid

Create a compact module grid with clear names and short explanations.

Modules:

- Digital Menu: premium mobile menus built for QR traffic and fast customer browsing.
- Mini-Games: lightweight branded games that turn customer attention into interaction.
- QR Experiences: campaign-specific pages, links, stickers, and customer entry points.
- Owner Intake: simple request flows for updates, project starts, and client communication.
- Command Center: future owner-facing dashboards for managing requests and brand modules.
- Website Concepts: premium web concepts that show how the local brand can evolve.
- Sticker/Signage Campaigns: physical-to-digital campaigns using QR stickers, counter signs, and seasonal prompts.

Keep the grid concise. It should feel like product modules, not a long services catalog.

## 6. Add Colattao Case Study Preview

Colattao should stay prominent as the first proof point.

Section purpose:

Show that Fina Calle OS already has a live reference combining a mini-game, menu, website concept, and owner-facing workflow.

Recommended copy:

Colattao Cafe Rush is the first live case study: a branded cafe experience with a mobile game, digital menu, website concept, and owner update flow.

Recommended CTAs:

- View Colattao
- View Menu
- Play Cafe Rush

Do not bury Colattao below generic feature copy. It is the evidence that makes the page credible.

## 7. Add Founder Pilot Section

Founder Pilot should position early clients as selected build partners.

Section goal:

Create a premium early-client lane without publishing fixed pricing or implying a discount.

Suggested language:

Founder Pilot clients work directly with Fina Calle to shape the first wave of interactive storefront modules. Each build is scoped around the business, the customer journey, and the modules that create the most immediate value.

Suggested details:

- Limited early-client availability
- Direct founder involvement
- Custom module mix
- Built to become repeatable Fina Calle OS patterns

## 8. Add No-Liability Payment Philosophy

The page should explain payment boundaries clearly without sounding defensive.

Principles to include:

- Client POS stays separate.
- Fina Calle does not process customer payments for the business in early builds.
- No Stripe, Square, cart, checkout, or stored-card logic on the landing page.
- Future Fina Calle client billing can use hosted payment links for AMMA/Fina Calle invoices.
- The system is designed to reduce unnecessary payment liability.

This should reassure business owners that the product can improve presentation and engagement without disrupting their existing payment system.

## 9. Add CTA to Start Intake

The current "Start a Project" CTA should evolve into a simple intake path.

Near-term CTA:

Start a Project

Future CTA:

Start Intake

Recommended intake positioning:

Tell us your business type, current customer touchpoints, and which modules you want to explore.

Do not add a full backend intake system until the intake structure is ready. A mailto or simple static inquiry path is acceptable for the next version.

## 10. Add Future Game Package Library Section

Add a forward-looking section for mini-game packages, without overpromising live inventory.

Section purpose:

Show that Fina Calle OS can create reusable mini-game engines for different business categories.

Examples:

- Conquest-style brand control games
- Cafe rush games
- Fuel and coffee rush games
- Barber and grooming games
- Restaurant order-flow games
- Gym discipline games

Keep this framed as a future library or product direction, not as a fully available catalog unless the games exist.

## 11. What Not To Do

Do not make it generic agency copy.

Avoid language like:

- We build beautiful websites
- We help brands grow online
- Full-service digital agency

Do not overprice yet.

Keep pricing flexible while the Founder Pilot and package boundaries mature.

Do not add payment logic.

No Stripe SDK, Square SDK, checkout, cart, payment form, card storage, or customer payment handling.

Do not promise Square/Stripe integrations as live.

Only mention future hosted payment links for Fina Calle client billing if needed.

Do not bury Colattao as the proof.

Colattao is the first live case study and should remain a first-scroll or early-page proof point.

## 12. Recommended Next Page Structure

Recommended order:

1. Hero: existing headline, subline, CTAs, Fina Calle OS identity
2. Colattao case study preview
3. What Fina Calle OS Builds
4. Module grid
5. Founder Pilot
6. No-liability payment philosophy
7. Future game package library
8. Start Intake CTA

This keeps the current brand voice while making the offer more concrete and ready for future expansion.
