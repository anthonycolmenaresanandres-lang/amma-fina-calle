# Character Identity Lock Protocol

## Purpose

Create strict, reusable identity contracts for recurring AI-generated characters. The locked text block is the primary continuity DNA. Reference images support the lock, but the text controls continuity.

## Absolute Rules

- Never rewrite, summarize, shorten, beautify, or casually improve an approved identity lock.
- An approved identity lock requires a version number, approval status, and approval note.
- Any change requires a new version number and a reason for change.
- Do not use synonyms for approved facial descriptors unless a new version is created. Synonym drift is identity drift.

## Strict Template

```txt
CHARACTER NAME:
[Name or role]

PROJECT:
[Project / campaign / client]

VERSION:
[v1.0]

APPROVAL STATUS:
Draft / Approved / Deprecated

FACE LOCK:
[Face shape, jaw, cheekbones, nose, mouth, asymmetry, scars, age markers]

BUILD LOCK:
[Height/frame, build, shoulders, posture, silhouette]

SKIN/TEXTURE LOCK:
[Skin tone, texture, weathering, marks, dirt, sweat, scars, realism]

HAIR LOCK:
[Hair color, hairline, length, texture, style]

EYES LOCK:
[Eye color, shape, depth, expression, eyebrow relationship]

FACIAL HAIR LOCK:
[None / stubble / beard shape / grooming level]

COSTUME LOCK:
[Wardrobe pieces, material, color, wear level, historical/brand accuracy]

EXPRESSION LOCK:
[Default expression, emotional restraint, mouth/eye behavior]

PERSONALITY ENERGY LOCK:
[Energy the character always carries]

LIGHTING COMPATIBILITY NOTES:
[Lighting that supports identity; lighting that risks changing identity]

NEGATIVE IDENTITY BLOCK:
[Everything this character must not become]

MOTION TEST PROMPT:
[Short micro-motion prompt for stress testing]

APPROVED USE CASES:
[Campaign video, menu hero, owner spokesperson, game skin, etc.]

FORBIDDEN CHANGES:
[Traits that cannot change without a new version]
```

## Approval Checklist

Before approving a lock:

- Face descriptors are specific and stable.
- Build descriptors preserve body silhouette.
- Costume/material identity is clear.
- Expression/personality identity is clear.
- Negative identity block is strong enough to prevent drift.
- Micro-motion test prompt is included.
- Forbidden changes are explicit.
