# Colattao Media Production Flow

## Purpose

This document defines the operating flow for Colattao Encanto Launch and future Fina Calle OS campaign production.

The flow separates approved inputs, normalized outputs, storyboard planning, provider shot packets, overlay assembly, and final export.

## 1. Input Assets

Required input categories:

- Brand assets: approved Colattao logo and Fina Calle mark if used.
- QR assets: approved QR PNG and verified destination.
- Digital Menu assets: approved screenshot, panel, or Digital Menu proof crop.
- Owner references: approved owner photo packet if owner appears.
- Cafe environment: approved environment packet if real cafe scenes appear.
- Product assets: approved drink/product/sticker assets.
- Background assets: approved campaign backgrounds.
- CTA copy: approved editable overlay text.

## 2. Required Approvals

Before production:

- Asset registry records must exist.
- Candidate assets must be promoted to approved or intentionally waived.
- Owner identity lock must be approved for owner-led clips.
- Cafe environment lock must be approved for cafe-specific clips.
- QR must be scan-tested.
- Digital Menu terminology must be preserved.

## 3. Normalization Outputs

Output packet:

- 9:16 background exports.
- 4:5 social crop exports.
- 1:1 sticker/product exports.
- Transparent logo overlays.
- Transparent CTA overlays if visual CTA art is used.
- Digital Menu proof crop.
- QR overlay PNG.
- Thumbnail stills.

## 4. Storyboard Packet

Each campaign needs:

- Campaign name.
- Audience.
- Core conversion target.
- 3 to 5 shot sequence.
- Shot objective.
- Product/Digital Menu proof moment.
- CTA moment.
- End-card moment.

## 5. Video Shot Packet

Each shot needs:

```txt
SHOT ID:
ASPECT RATIO:
DURATION:
SOURCE ASSETS:
SUBJECT:
BACKGROUND:
CAMERA:
MOTION:
ONE MOTION RULE:
Z-AXIS BLOCKING:
OVERLAYS:
PROVIDER TARGET:
OUTPUT COUNT:
ESTIMATED COST:
QA RULES:
APPROVAL STATUS:
```

## 6. Overlay Packet

Overlay packet includes:

- Logo overlay.
- QR overlay.
- CTA overlay.
- Digital Menu proof overlay.
- Captions if used.
- Footer/signature overlay.
- Safe-area placement map for 9:16.

Overlay rules:

- Text remains editable.
- QR remains scan-tested.
- Logo remains approved source art.
- Digital Menu text is not regenerated.

## 7. Final Assembly Packet

Final packet includes:

- Final exported 9:16 video.
- Optional 4:5 version.
- Optional square thumbnail.
- Source stills.
- Overlay source files.
- Provider outputs.
- QA report.
- Approval record.

## 8. QA Gate

Reject output if:

- Owner likeness drifts.
- Cafe environment drifts.
- Product shape changes materially.
- Any logo is generated or mutated.
- Any QR is generated or broken.
- Any Digital Menu text is generated or wrong.
- Any CTA text is generated or misspelled.
- Wrong terminology appears.
- Motion is chaotic or off-brand.
- Mobile readability fails.

## 9. Registry Writeback

After QA, update or create registry records for:

- Normalized assets.
- Provider-generated stills.
- Provider-generated video clips.
- Final overlays.
- Final exports.
- Rejected outputs.
