# Cafe Environment References

## Rule

Cafe references must be approved before environment lock. Do not generate a campaign cafe, restaurant, storefront, or owner environment from memory.

## Required Cafe Reference Record

```txt
ASSET ID:
ASSET NAME:
SOURCE:
DATE ADDED:
STATUS: approved / pending / rejected / deprecated
INTENDED USE:
FORBIDDEN USE:
REPLACEMENT RULE:
OWNER / APPROVER:
FILE PATH / LOCATION:
REFERENCE TYPE: counter / wall / seating / exterior / menu board / lighting / props / other
ENVIRONMENT LOCK USED BY:
NOTES:
```

## Minimum Cafe Reference Set

Before creating a cafe environment lock, collect:

- Counter layout.
- Wall color/material.
- Lighting temperature.
- Menu placement.
- Owner position.
- Customer position if customers appear.
- Props.
- Approved camera angles.

## Replacement Rule

If the cafe environment changes or a campaign uses a different area of the cafe, create a new environment reference record and update the environment lock version.
