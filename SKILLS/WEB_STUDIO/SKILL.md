# Web Studio

## Core Workflow

1. Complete strategy intake before design or build.
2. Produce approved visual sections before production code.
3. Build an overlay link map for clickable regions.
4. Create SEO/copy layer from approved facts only.
5. Follow preview/production workflow in `HANDOFFS/PREVIEW_PRODUCTION_REVIEW_WORKFLOW.md`.
6. Use approved logos only.

## Required Outputs

- Strategy intake status.
- Approved visual section list.
- Overlay link map.
- SEO/copy layer.
- Protected files.
- Build/QA checklist.

## Guardrails

- No app code changes from planning tasks.
- No prices, hours, offers, claims, or business facts without approval.
- No auth/payment/backend/database additions unless explicitly approved.
- No AI-generated final logos.

## Starter Protocol

Use `STARTER_PROTOCOL.md` in this module for the first build packet format.
