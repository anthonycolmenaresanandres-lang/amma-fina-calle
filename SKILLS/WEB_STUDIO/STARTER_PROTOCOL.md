# Web Studio Starter Protocol

## Inputs

- Client name and slug.
- Strategy intake answers.
- Approved desktop sections.
- Approved mobile sections.
- Brand colors/fonts/tone.
- Overlay link map.
- SEO/copy layer.
- Approval record.

## Output Packet

```txt
CLIENT:
STAGE: preview / production
APPROVED FACTS:
PENDING FACTS:
VISUAL SECTIONS:
OVERLAY LINKS:
SEO COPY:
PROTECTED FILES:
BUILD COMMAND:
QA REQUIREMENTS:
```

## Review Rule

If the visual design is not approved, do not code. If the facts are not approved, use placeholders or omit the claim.
