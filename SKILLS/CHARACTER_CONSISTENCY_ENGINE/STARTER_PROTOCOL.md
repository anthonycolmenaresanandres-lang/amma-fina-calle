# Character Consistency Starter Protocol

## Inputs

- Client or brand name.
- Character role.
- Existing image references or description.
- Required usage: website, campaign, game, sticker, menu, or social.
- Allowed variations.
- Forbidden changes.

## Process

1. Draft the identity lock.
2. Draft the negative identity.
3. Generate only preview candidates.
4. Compare each candidate against the lock.
5. Promote only the strongest candidate to production review.

## Output Packet

```txt
CHARACTER:
IDENTITY LOCK:
NEGATIVE IDENTITY:
ALLOWED VARIATIONS:
REJECT IF:
REVIEW STATUS:
```
