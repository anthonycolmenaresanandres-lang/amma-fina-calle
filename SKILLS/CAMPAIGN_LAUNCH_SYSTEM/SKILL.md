# Campaign Launch System

## Core Workflow

1. Define the campaign outcome.
2. Choose the module stack: Digital Menu, mini-game, website concept, QR/signage, stickers, or owner request.
3. Apply character, cinematic, and prompt rules where relevant.
4. Build preview assets first.
5. Run preview/production review before app changes or launch.
6. Record handoff notes in `HANDOFFS/HANDOFF_TEMPLATE.md` format.

## Required Outputs

- Campaign brief.
- Asset list.
- Shot matrix if visuals are needed.
- Prompt packet.
- Review checklist.
- Production scope.
- Launch handoff.

## Guardrails

- No fake campaign claims.
- No unapproved pricing or offers.
- No payment/auth/backend/database work by default.
- No AI-generated logos.
- No deployment changes unless explicitly authorized.

## Starter Protocol

Use `STARTER_PROTOCOL.md` for launch packet structure.
