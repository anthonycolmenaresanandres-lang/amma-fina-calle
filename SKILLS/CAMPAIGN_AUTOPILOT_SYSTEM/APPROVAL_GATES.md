# Campaign Autopilot Approval Gates

## Purpose

These gates prevent Fina Calle OS campaign automation from turning incomplete or unsafe inputs into client-facing outputs.

## Gate 0: Scope Gate

Stop if:

- The requested folder is outside approved scope.
- The task would modify app code, routes, deployments, configs, package files, or secrets.
- The task would touch an unrelated client repo.

Required approval:

- Anthony confirms the target workspace and client asset folder.

## Gate 1: Asset Source Gate

Stop if:

- An asset path is missing or invented.
- A logo, QR, menu screenshot, or owner reference is not traceable to a source.
- Asset status is unknown and the workflow would treat it as approved.

Required approval:

- Source and status are recorded in the asset registry.

## Gate 2: Logo / QR / Text Overlay Gate

Stop if:

- Logo would be AI-generated or redesigned.
- QR would be generated by AI, redrawn, or altered from a real source.
- CTA, menu copy, pricing, or legal text would be baked into generated imagery.

Required approval:

- Logo overlay approved or candidate-labeled.
- QR scan-tested and destination recorded.
- CTA text approved as editable overlay.

## Gate 3: Owner Identity Gate

Stop if:

- Owner/spokesperson appears prominently but references are insufficient.
- Front/3/4/profile/full-body references are missing and no waiver exists.
- The output implies final identity lock without approval.

Required approval:

- Owner identity lock is approved, or owner use remains internal/draft only.

## Gate 4: Venue Environment Gate

Stop if:

- Environment references are missing.
- Background would be invented as the venue.
- A low-quality or generated venue asset would be treated as final.

Required approval:

- Venue references are approved or intentionally waived.

## Gate 5: Digital Menu Proof Gate

Stop if:

- Campaign claims a live Digital Menu proof without an approved screenshot/crop.
- Text, prices, or menu items would be invented.
- Colattao campaign language uses forbidden terminology.

Required approval:

- Digital Menu screenshot/proof and destination are registered.

## Gate 6: Spend / Provider Gate

Stop if:

- Any paid API/provider call would run.
- Any secret key or provider credential is required.
- Cost estimate has not been reviewed.

Required approval:

- Anthony explicitly approves provider, budget, and run scope.

## Gate 7: Client-Facing Final Gate

Stop if:

- Output is still a candidate/review draft.
- Visual QA is incomplete.
- QR/client CTA/logo approval is missing.
- File naming/status does not mark final approval.

Required approval:

- Human approval recorded in a review packet.

## Gate 8: Publishing / Deployment Gate

Stop if:

- The workflow would deploy, post, publish, email, or push client-facing assets.

Required approval:

- Separate explicit publish/deploy instruction from Anthony.
